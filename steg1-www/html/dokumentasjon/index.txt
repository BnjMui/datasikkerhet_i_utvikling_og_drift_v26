Endre passord – funksjonalitet
Det er utviklet en side for endring av passord i Emneportalen. Løsningen er laget med PHP, HTML og CSS, og bruker sesjonsbasert innlogging.
Brukeren kan skrive inn nytt passord og bekrefte dette i et skjema. Systemet sjekker at passordene matcher og oppfyller minimumskrav før funksjonen for passordendring kjøres. Ved vellykket endring blir brukeren sendt tilbake til forsiden. Eventuelle feilmeldinger vises på siden.

Teknologi og struktur
•	PHP for serverlogikk
•	Session-håndtering (session_start)
•	Ekstern tjenestefil for passordendring
•	Semantisk HTML (main, section, fieldset)
•	Gjenbruk av header.php og footer.php
•	Egen CSS for styling

Src > sql > init.sql
Dette er SQL-skriptet som oppretter databasen og tabellene som brukes i Emneportalen.
Hva som er laget
Databasen består av følgende hovedtabeller:
•	users
 Lagrer alle brukere (student og foreleser) med felles felter som navn, e-post, rolle og passord.
•	students
 Inneholder student-spesifikke data (studieretning og studiekull).
 Koblet til users via fremmednøkkel.
•	lecturers
 Inneholder foreleser-spesifikke data (profilbilde og sikkerhetsspørsmål).
 Koblet til users via fremmednøkkel.
•	courses
 Lagrer emner med emnekode, emnenavn og PIN-kode.
 Koblet til foreleser.
•	students_courses
 Koblingstabell mellom studenter og emner (mange-til-mange-relasjon).
•	messages
 Meldinger sendt av studenter til et emne.
•	replies
 Ett svar per melding (typisk fra foreleser).
•	comments
 Kommentarer knyttet til meldinger.
•	reports
 Rapportering av meldinger.
Struktur og relasjoner
•	Bruk av primærnøkler (PRIMARY KEY) og fremmednøkler (FOREIGN KEY).
•	Bruk av ON DELETE CASCADE for å sikre at tilknyttede data slettes automatisk.
•	Bruk av:
o	AUTO_INCREMENT for ID-er
o	ENUM for rolle (student/lecturer)

Backend-støtte for Frontend
Denne delen av prosjektet består av PHP-filer som kobler frontend til backend-API-et. Filene fungerer som et mellomledd mellom brukergrensesnittet og API-endepunktene.
 
Oversikt over filer
config.php:
Inneholder grunnleggende konfigurasjon for prosjektet. Definerer API-porten (8001) og base-URL for alle API-kall. Denne filen gjør det enkelt å endre API-adressen på ett sted.
api_client.php:
Håndterer all kommunikasjon med backend-API-et. Bruker CURL for å sende HTTP-forespørsler (GET, POST) til localhost:8001/api og returnerer JSON-svar. Denne filen er bindeleddet mellom frontend og backend. 
login_api_service.php:
Inneholder funksjoner for brukerautentisering og registrering:
-	get_login() — logger inn brukere (studenter og forelesere)
-	register_student() — registrerer nye studenter
-	register_lecturer() — registrerer nye forelesere med emne og PIN-kode
-	change_password() — bytter passord for innlogget bruker
-	get_security_question() — henter sikkerhetsspørsmål ved glemt passord
-	forgot_password() — tilbakestiller passord via sikkerhetsspørsmål

course_api_service.php:
Håndterer all emne- og meldingsrelatert funksjonalitet:
-	get_course() — henter et spesifikt emne med meldinger (støtter PIN-kode for gjester)
-	get_courses() — henter alle emner
-	create_message() — oppretter ny melding fra student
-	create_reply() — oppretter svar fra foreleser
-	create_comment() — legger til kommentar fra gjest
-	create_report() — rapporterer upassende melding

header.php:
Gjenbrukbar header-komponent som vises øverst på alle sider. Viser innlogget brukers navn og rolle (student/foreleser), eller en innloggingsknapp for gjester. Bruker $_SESSION['session_data'] for å sjekke om noen er logget inn.
footer.php:
Gjenbrukbar footer-komponent med kontaktinformasjon.
compose.yaml:
Docker-konfigurasjon for å kjøre MySQL-database lokalt under utvikling. 
 
Hvordan frontend bruker backend
1.	Konfigurasjon: config.php definerer hvor API-en ligger
2.	API-kommunikasjon: api_client.php sender alle HTTP-forespørsler til backend
3.	Service-lag: login_api_service.php og course_api_service.php pakker inn API-kall i enkle funksjoner
4.	Frontend: HTML-sider kaller service-funksjonene i stedet for å snakke direkte med API-et
 
Teknologier brukt
-	PHP  Serverside-logikk
-	CURL  HTTP-forespørsler til API
-	Sessions  Brukerautentisering (lagrer session_data)
-	JSON  Dataformat mellom frontend og API
-	Docker + MySQL  Database (under utvikling)
